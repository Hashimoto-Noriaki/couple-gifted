# Claude Code スキルの作り方

Claude Code の `/スキル名` コマンドは、`.claude/skills/<スキル名>/SKILL.md` を置くだけで追加できる。

## ディレクトリ構成

```
.claude/
  skills/
    <スキル名>/
      SKILL.md   ← スキルの指示書（これだけでOK）
```

## SKILL.md の書き方

```markdown
---
description: このスキルが何をするかの一行説明
---

1. 最初にやること
2. 次にやること
3. ...

## 補足セクション（任意）

- ルールや注意点を書く
```

- フロントマターの `description` はスキル一覧に表示される
- 手順は番号付きリストで書くと Claude が順番通りに実行する
- 「ユーザーに確認してから実行する」と書けば確認ステップを挟める

## 実例: pr-description スキル

```
.claude/skills/pr-description/SKILL.md
```

```markdown
---
description: PRの説明文をテンプレートに沿って生成する
---

1. ベースブランチを特定する
2. git diff で差分を確認する
3. テンプレートに沿って PR 説明文を生成し、ユーザーに提示する
4. ユーザーの承認を得たら gh pr create でPRを作成する
```

## 他のアプリへの導入手順

1. `.claude/skills/<スキル名>/` ディレクトリを作る
2. `SKILL.md` を上記フォーマットで作成する
3. Claude Code で `/スキル名` と入力すれば即座に使える（再起動不要）

## よく使うスキルのパターン

| やりたいこと | 指示の書き方 |
|---|---|
| コマンド実行前に確認 | `ユーザーの承認を得てから <コマンド> を実行する` |
| ファイルを読んで処理 | `<ファイルパス> を読み込み、〜する` |
| 既存ファイルのフォーマットに従う | `<ファイルパス> のフォーマットに沿って生成する` |
| 条件分岐 | `〜の場合は〜し、そうでなければ〜する` |
