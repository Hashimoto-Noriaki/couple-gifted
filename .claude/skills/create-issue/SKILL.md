ユーザーの説明をもとに GitHub issue を作成する。

## 手順

1. issue の種別を確認する（バグ報告 / 機能要望 / タスク）
2. タイトルと詳細をユーザーと決める
3. 以下のテンプレートで本文を組み立てる
4. `gh issue create --title "タイトル" --body "本文"` で作成する

## 本文テンプレート

**バグ報告:**
```
## 再現手順
1. 

## 期待する動作


## 実際の動作

```

**機能要望 / タスク:**
```
## 背景・目的


## 実装したい内容


## 完了条件
- [ ] 
```

## ラベル

変更内容に応じて `--label` を付与する:
- `bug` — バグ報告
- `enhancement` — 機能要望
- `chore` — 設定・ドキュメント系
