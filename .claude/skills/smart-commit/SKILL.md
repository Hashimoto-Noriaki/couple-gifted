---
description: ステージ済みの差分からコミットメッセージを生成する
---

## 手順
1. `git diff --cached` を実行してステージ済みの差分を確認する
2. ステージされたファイルがない場合は、`git status` を実行して確認を促す
3. 差分の内容に基づいて適切なプレフィックスとメッセージを生成する
4. ステージされていないファイルがある場合は一覧を表示する

## プレフィックス規則

| prefix | 用途 |
|--------|------|
| feat   | 新機能追加 |
| fix    | バグ修正 |
| test   | テストのみの追加・修正 |
| refactor | リファクタリング |
| docs   | ドキュメント更新 |
| chore  | 設定・ツール変更 |

## 注意

実装ファイルとテストファイルが混在している場合は、原則として `feat:` (または `fix:`) と `test:` の 2 つのコミットに分割して提案する
- ただし、テストファイルの変更が軽微な場合（タイポ修正など）は分割不要と判断してもよい
- メッセージは 1 行で完結させる（本文不要）
- ステージされていないファイルがある場合は一覧を表示して確認を促す

## 例

**良い例:**
- `feat: ユーザー登録時のメール認証機能を追加`
- `fix: ログイン時の CSRF トークン検証エラーを修正`
- `test: User モデルのバリデーションテストを追加`
- `chore: RuboCop の設定を更新`

**避けるべき例:**
- `feat: 色々修正`（具体性がない）
- `fix: バグ修正しました。詳細は以下の通りです...`（複数行・冗長）
- `update`（規定外のプレフィックス）
